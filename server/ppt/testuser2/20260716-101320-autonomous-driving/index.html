DEFINITION · L0–L5

什么是自动驾驶？——分级标准 L0–L5

SAE J3016（2021）与国标 GB/T 40429-2021 基本对应；核心维度：驾驶操作、环境监测、后备响应、适用场景（ODD）。
L0–L2 · 驾驶辅助

L0 应急辅助 / L1 部分辅助 / L2 组合驾驶辅助

系统协助人类，不能独立承担驾驶任务；驾驶员为责任主体（ACC、LKA、高速巡航+车道保持）。

L3 · 有条件自动驾驶

系统持续执行驾驶任务

在 ODD 内自主行驶，人类需应请求接管；责任开始从驾驶员向系统转移。

L4–L5 · 自动驾驶

L4 高度 / L5 完全自动驾驶

L4 在 ODD 内完全自主；L5 任意场景无 ODD 限制，系统为主责。

0–2 级 = 驾驶辅助（人主导）   3–5 级 = 自动驾驶（系统主导）

ARCHITECTURE · 三支柱

如何「看懂」世界？——技术架构三支柱

① PERCEPTION

感知 Perception

摄像头 / 激光雷达 LiDAR / 毫米波雷达 / 超声波雷达；多传感器融合（卡尔曼滤波、Transformer 跨模态注意力）。

② PLANNING

决策规划 Planning

预测障碍物轨迹 → 全局路径(A*/Dijkstra) + 局部规划(Lattice/RRT/MPC) → 行为决策。

③ CONTROL

执行控制 Control

线控底盘：纵向 PID + 横向 Pure Pursuit / MPC，将决策精确转化为油门、制动、转向。

LOCALIZATION

定位与融合

RTK-GNSS+IMU 组合、激光/视觉 SLAM、高精地图辅助，实现厘米级定位。

SOFTWARE

软件架构路线

模块化 Pipeline（可解释强）与端到端 E2E（误差不传播）两条路线并存演进。

EVOLUTION · 2025 HOT

从规则到大模型——算法演进路线

2025 年技术热点：BEV+Transformer → Occupancy → 模块化端到端 → One Model → VLA / 世界模型。
STAGE 1

模块化 Pipeline

感知/预测/规划/控制分模块独立优化，接口结构化；可解释性强、易调试，但误差逐级传播。

STAGE 2

端到端 E2E

BEV+Transformer 统一空间表征 → Occupancy 体素占据；单网络从感知直出轨迹，规避模块间对齐损耗。

STAGE 3 · 2025

VLA / 世界模型

视觉-语言-动作一体化（如理想 MindVLA）；用大模型重塑架构，迈向可推理的类人驾驶。

代表：Tesla FSD v12 UniAD 理想 MindVLA Waymo World Model

CHALLENGES · 三线待破

核心挑战——技术、责任与成本

技术 · 长尾

长尾场景

极端天气、临时施工、无保护左转等难例，仍是算法泛化的硬骨头。

责任 · 界定

归责转移

L3 事故归责从「驾驶员」转向「系统算法」，数据黑箱待破解。

数据 · 壁垒

隐私与网安

跨境数据限制；通信协议、OTA 升级面临系统性攻击风险。

成本 · 悬崖

L3 硬件冗余

双制动/双转向/高算力芯片使整车成本增加 5–10 万元。

破局点

三线齐破

安全数据公开建立信任 · 责任框架与保险配套 · 全行业降本共识。
